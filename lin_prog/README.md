# Лабораторная работа - Линейное Программирование в Задачах Оптимизации
## Теоретическая часть  

Линейное программирование (ЛП) — это раздел математического программирования, посвященный поиску экстремумов (максимумов или минимумов) линейных целевых функций при наличии системы линейных ограничений.

Основные компоненты модели:

* Целевая функция: Линейная функция, значение которой необходимо максимизировать (например, прибыль) или 
минимизировать (затраты).

* Система ограничений: Линейные равенства или неравенства, определяющие область допустимых решений.

* Условия неотрицательности: Переменные решения обычно должны быть больше или равны нулю.

Для решения задач ЛП в Python используется модуль scipy.optimize, в частности функция linprog, реализующая симплекс-метод и методы внутренней точки.

## Задача 1
Постановка задачи:  
Необходимо максимизировать прибыль предприятия, выпускающего смартфоны (x_1) и планшеты (x_2).  

Параметры:  

* Прибыль: 8000 за смартфон, 12000 за планшет.  
* Ресурс 1 (Процессоры): 2x_1 + 3x_2 <= 240  
* Ресурс 2 (Память): 4x_1 + 6x_2 <= 480  
* Ресурс 3 (Аккумуляторы): 1x_1 + 2x_2 <= 150  
* Условие: x_1, x_2 >= 0  

Математическая модель и функция Лагранжа:  
  
Для использования в linprog нужно минимизировать отрицательную прибыль.   
Целевая функция: f = -(8000x_1 + 12000x_2)    
Функция Лагранжа (в упрощенном для анализа виде): L = -(8000x_1 + 12000x_2) + L1*(2x_1 + 3x_2 - 240) + L2*(4x_1 + 
6x_2 - 480) + L3*(x_1 + 2x_2 - 150) Где L1, L2, L3 — множители Лагранжа (теневые цены ресурсов).  
  
Результат:  

* Интерпретация: Оптимальный план — производить 30 смартфонов и 60 планшетов (или иная комбинация на границе, так как ограничения 1 и 2 зависимы). Максимальная прибыль составит 960 000 руб.  
* Использование ресурсов: Процессорное время и память используются полностью (активные ограничения). Аккумуляторы остаются в избытке (30 шт.).  
* Чувствительность: Наиболее дефицитным ресурсом является процессорное время, так как его увеличение напрямую ведет к росту прибыли.

## Задача 2

Постановка задачи:  
Требуется минимизировать затраты на перевозку МТО с двух складов (S1, S2) на три базы (Альфа, Бета, Гамма).  
  
Математическая модель:   
* Переменные: x_ij — объем перевозки со склада i на базу j  
* Целевая функция: Z = 8x_11 + 6x_12 + 10x_13 + 9x_21 + 7x_22 + 5x_23 -> min  
* Ограничения-равенства:   
        Склады: x_11+x_12+x_13 = 150; x_21+x_22+x_23 = 250  
        Базы: x_11+x_21 = 120; x_12+x_22 = 180; x_13+x_23 = 100  
  
  
Баланс задачи:  

* Запасы: 150 + 250 = 400 т.  
* Потребности: 120 + 180 + 100 = 400 т.   
* Вывод: Задача сбалансирована (закрытого типа).  
  
Результат:  

Оптимальный план:  
* Склад 1 -> База Бета: 150 т.  
* Склад 2 -> База Альфа: 120 т.  
* Склад 2 -> База Бета: 30 т.  
* Склад 2 -> База Гамма: 100 т.  

Стоимость: 2450 усл. ед.   

Логистика: Все базы удовлетворены, склады пусты. Маршрут Склад 1-> База Гамма не используется из-за высокой стоимости (10) по сравнению с поставкой со Склада 2 (5).
